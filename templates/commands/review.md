# /review - 课程质量评估系统 ⭐

> **核心理念**: 对课程内容进行全面的质量评估
> **目标**: 发现问题,提供改进建议,确保课程质量

---

## 第一步: 运行脚本获取状态 ⚠️ 必须执行

```bash
bash scripts/bash/review.sh
```

脚本返回:
- `spec`: 课程规格
- `has_outline`: 是否有大纲
- `has_chapters`: 是否有章节内容
- `chapters_count`: 章节数量

---

## 第二步: 检查参考课程对标 ⚠️ 可选

### 检查是否有参考课程库

如果存在 `reference-courses/index.json`:

**提示用户**:

```
📊 质量对标选项

发现参考课程库,可以将你的课程与行业标杆对比。

是否启用质量对标? (y/n)
  y - 与相似专栏对比,获得改进建议
  n - 仅进行基础质量评估
```

**⚠️ 等待用户选择**

如果用户选择 `y`,则在评估报告中增加对标部分。

---

## 第三步: 检查激活的作者风格 ⚠️ 必须执行

### 检查是否激活了作者 Persona

```bash
# AI 操作: 检查是否有激活的作者风格
if [ -f ".courseify/active-persona.yaml" ]; then
  echo "发现激活的作者风格"
fi
```

### 如果存在激活的 Persona

**读取并应用作者风格**:

```bash
# AI 操作: 读取 persona 配置
Read: .courseify/active-persona.yaml
```

**从 YAML 中提取**:
- `author.name` - 作者名
- `author.course` - 课程名
- `facilitation_prompts.when_reviewing_quality` - 质量评估提示词

**应用作者风格**:

```
🎭 检测到激活的作者风格: [author.name] (《[course]》)

在评估质量时,我将参考该作者的质量标准:

[显示 facilitation_prompts.when_reviewing_quality 的内容]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 提示: 我会用该作者的质量标准来评估你的课程,
         指出与该作者风格的差距和改进方向。
```

**注意**: AI 在后续评估过程中,必须:
1. 应用该作者的质量标准进行评估
2. 指出与该作者风格的差距
3. 提供符合该作者风格的改进建议

### 如果没有激活 Persona

使用默认的通用质量标准评估。

## 评估标准说明

### 清晰度 (Clarity)
- 标题是否清晰
- 目标是否明确
- 描述是否易懂

### 结构性 (Structure)
- 逻辑是否连贯
- 难度是否递进
- 体系是否完整

### 吸引力 (Engagement)
- 内容是否有趣
- 案例是否生动
- 形式是否多样

### 实用性 (Practicality)
- 是否贴近实际
- 是否有实践价值
- 是否能学以致用

### 完整性 (Completeness)
- 知识点是否完整
- 练习是否充足
- 资源是否齐全
